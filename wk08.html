From this lesson, I learnt about:


  1. Script Placement in Web Pages:
    • To optimize page rendering, it is suggested to place JavaScript code at the end of the page, just before the closing tag.
    • This way, the browser can continue loading and rendering other page elements while simultaneously fetching and executing the JavaScript code.
  2. Key Attributes:
    • HTML5 allows custom attributes to be created data-(custom name)
    • The data attribute let you stores information in a HTML element.
    • The name of the data is whatever that comes after the dash
  3. Get & Set: Attribute
    • Getting: document.querySelector('.highlight').getAttribute('data-color'); //returns "green"
    • Setting: document.querySelector('.highlight').setAttribute('data-color','red');
  4. Live NodeLists:
    • Updated in real-time as the script modifies the page.
    • Using methods such as getElementById(), getElementsByClassName()
    • Dynamic and reflect changes made by the script.
    • Generally faster to generate.
  5. Static NodeLists:
    • Not updated to reflect changes made by the script.
    • Result from methods like querySelector() and querySelectorAll().
    • Remain unchanged after the initial query.
    • Reflect the document state at the time of the query.
  1. Introduction to AJAX:
    • AJAX stands for Asynchronous JavaScript and XML.
    • Emphasized as not being a framework, library, or technology on its own.
  2. Definition of AJAX:
    • AJAX is a technique for loading data into part of a webpage without requiring a full page refresh.
    • Commonly uses JSON (JavaScript Object Notation) for data format.
  3. Benefits of AJAX:
    • Improves user experience by loading new content into a page selectively.
    • Enables partial updates, avoiding the need for users to wait for an entire page to load.
    • Form Events: Events associated with form elements.
  4. Examples of AJAX in Action:
    • Live search or autocomplete, such as in Google search, is cited.
    • Websites with user-generated content like Twitter and Flickr utilize AJAX to display dynamic information.
  5. Why Use AJAX:
    • Contrasts the synchronous processing model of browsers (halting page processing) with AJAX's asynchronous (non-blocking) processing model.
    • Allows the browser to request data from a server while continuing to load the rest of the page.
  6. Using AJAX While Pages Are Loading:
    • AJAX enables updating specific parts of a page without requiring a full page refresh.
    • Explains how an Ajax engine, written in JavaScript, facilitates communication between the user and the server.
  7. AJAX Simplified:
    • Describes AJAX as a technology for building Single Page Applications (SPAs) like Twitter, Facebook, Gmail, etc.
    • Illustrates how AJAX allows webpages to request and process data from a server in the background without disrupting the current page.
  8. How AJAX Works:
    • Outlines the steps involved in making an AJAX request: creating an XMLHttpRequest, creating a callback function, opening a request, and sending the request.
    • Differentiates between HTML, XML, and JSON as data formats used in AJAX.
  9. Drawbacks of Different Data Formats:
    • Highlights benefits and drawbacks of HTML, XML, and JSON in terms of ease of use, data portability, and processing.
  10. Introduction to APIs:
    • Defines API as an Application Programming Interface, providing additional functionality and features for web developers to integrate into websites.
    • Emphasizes benefits for end users in terms of customization, flexibility, and quicker interactions.
