RESTDB & NOSQL

From this lesson, I learnt about:


  1. SQL serves as the backbone of relational databases, offering a robust framework for accessing and managing structured data. It operates primarily with tables and enforces relationships between them.
  2. Structured Data Format:
    • SQL databases organize data into tables with predefined schemas.
    • This structured approach ensures data integrity and facilitates efficient querying.
  3. Relationships:
    • SQL databases rely heavily on relationships between tables, maintained through foreign keys.
    • This allows for complex queries and ensures data consistency.
  4. Common Examples:
    • Popular SQL databases include MySQL, PostgreSQL, and Microsoft SQL Server.
  1. NoSQL databases depart from the traditional relational model, offering flexibility and scalability for handling unstructured or semi-structured data. They eschew rigid schemas and emphasize horizontal scalability.
  2. Unstructured Data Handling:
    • NoSQL databases excel at managing unstructured or semi-structured data, such as JSON documents.
    • They organize data into collections rather than tables, providing a more flexible data model.
  3. Dynamic Schema:
    • Unlike SQL databases, NoSQL databases feature dynamic schemas, allowing for agile development and accommodating evolving data requirements.
  4. Scalability:
    • NoSQL databases are designed for horizontal scalability, enabling them to handle massive volumes of data across distributed systems with ease.
  5. Examples:
    • Prominent NoSQL databases include MongoDB, Cassandra, RestDB and Redis.
  1. Data Model:
    • SQL databases employ a tabular structure with predefined schemas, whereas NoSQL databases use flexible document-oriented or key-value pair models.
    • It provides a way to persistently store key-value pairs locally on the client side.
  2. Query Language:
    • SQL databases utilize the SQL query language for data manipulation and retrieval, while NoSQL databases may have proprietary query languages or rely on APIs for interaction.
  3. Scalability and Performance:
    • NoSQL databases excel at horizontal scalability and can efficiently handle large volumes of data and high throughput, making them suitable for applications requiring real-time analytics or massive-scale data storage.
