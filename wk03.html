From this lesson, I learnt about:


  • Wireframe: Like a blueprint, conveying layout, functionality, content types, and information hierarchy for websites and applications.
  • Wireframes ARE NOT: Visual Design, Typography, Icongraphy, Photography
  • Wireframes CAN: Mind maps rapidly generate ideas, facilitate discussions, display content hierarchy, and establish essential functionality.
  • Static Wireframes:
    Pros: Usually quick to create | Variety of tools and software
    Cons: Not responsive
  • Low Fidelity Wireframes:
    Pros: Can quickly creating and brainstorming multiple ideas without immediate commitment, often in a simple sketch-like form.
    Cons: Might not have enough detail for every situation
  • High Fidelity Wireframes:
    Pros: High fidelity prototypes offer greater detail and closely resemble the final layout compared to low fidelity versions.
    Cons: Stakeholders might confuse with design
  • form element: used to create an HTML form for user input
  • input element: Can be displayed in many ways, depending on the type attribute:
    type="text" (Displays a single-line text input field)
    type="radio" (Displays a radio button (for selecting one of many choices))
    type="checkbox" (Displays a checkbox (for selecting zero or more of many choices))
    type="submit" (Displays a submit button (for submitting the form))
    type="button" (Displays a clickable button)
  • label element: The label element enhances accessibility by providing vocal cues for screen-reader users, aids users with small interactive elements by allowing text-click interactions, and requires the "for" attribute to be linked with the corresponding input element's "id."
  • footer element: defines a footer for a document or section
  • nav element: defines a set of navigation links
  • aside element: defines some content (indirectly related to the surrounding content) aside from the content it is placed in (like a sidebar)
  • id attribute: used to specify a unique id for an HTML element
  • class attribute: used to specify a class for a HTML element or MULTIPLE HTML elements that share the same class
  • Descendant Selector
  • Grouping Selector
  • Inheritance: CSS styles can be inherited from the parent elements
  • Specificity: Specificity determines how browsers decide which CSS rule takes precedence (Universal, type, class, id)
  • Cascade: Style declarations cascade and are read from top to bottom
  • !important: The !important keyword overrides source order and specificity (Added just before the end of the rule declaration before semicolon(;))
  • Media Queries: Used to modify the CSS of a site depending on quanlities such as viewport size or device type
  • @media: Applies one or more CSS properties based on the result of a media query
  • min-width: If the minimum width of the viewport is at least the specific width or wider, then the media query is true
  • max-width: If the maximum width of the viewport is at no more than the specified width, or it's narrower then the media query is true
  • Relative Position: Moves elements around relative to where they would appear on the page
  • Absolute Position: Offset is relative to the entire browser window instead of the original position of the element
  • Fixed Position: Fixed elements don't scroll with the rest of the page. Commonly known as sticky elements
  • Z-Index: z-index property lets you control the depth of elements on the page (z-index work ONLY on elements with a position property set to absolute, fixed, or relative)
  • Negative Z-Index: Go farther into the page (behind)
  • Positive Z-Index: Comes out of the page
  • Flex: Apply flexbox to get flexible layouts
