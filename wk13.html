Best Practices - Common Mistakes

From this lesson, I learnt about:


  1. Acknowledgement (README):
    • Acknowledge those who contributed to the project's success, including mentors, classmates, inspirations, idea contributors, testers, and knowledge sharers.
    • Add an acknowledgment section in the README.md file.
  2. Mixed Content Error:
    • Problem: Mixed content error when accessing an API with HTTP on HTTPS sites.
    • Solutions: Test API for HTTPS compatibility, use a CORS fix (e.g., cors-anywhere) for API endpoint, check API documentation and errors.
  3. API Testing Issues:
    • Problem: API works in Postman but not in code.
    • Solutions: Check API documentation, whitelist domains if required, inspect console and network logs for errors.
  4. RapidAPI Usage Issues:
    • Problem: Issues with free RapidAPI endpoints.
    • Solutions: Subscribe to endpoints, handle hard limits (e.g., using Postman, extracting JSON to a file, using AJAX to access local JSON).
  5. Chart JS Usage:
    • Question: Can Chart JS fulfill API requirements?
    • Guidelines: Ensure dynamic data usage, document rationale in README.
  6. CSS Framework Usage:
    • Question: Can Material.io or other frameworks be used?
    • Guidelines: Allowed, justify usage, ensure responsiveness, document design choices.
  7. Using External JSON Files:
    • Question: Can JSON files from other websites be used?
    • Guidelines: Allowed if publicly accessible, use proper AJAX techniques.
  8. API Failure Handling:
    • Problem: API failure.
    • Solutions: Rework idea or use alternative API, test in Postman, check documentation, handle CORS issues.
  9. Troubleshooting Tips:
    • Problem: Code not working.
    • Tips: Check console output, describe situation and attempts made when seeking help, avoid just pasting screenshots, provide code links.
