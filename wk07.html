From this lesson, I learnt about:


  1. DOM Tree Structure:
    • A web browser creates a model of a webpage known as the Document Object Model (DOM) tree.
    • The DOM tree consists of nodes representing elements, attributes, and text in an HTML document.
  2. Types of Nodes in DOM:
    • Document Node: Represents the entire page document.
    • Element Node: Represents HTML elements and describes the structure of the document.
    • Attribute Node: Represents attributes of HTML elements.
    • Text Node: Represents the text content within HTML elements.
  3. Accessing the DOM:
    • To access the DOM tree, start by looking for element nodes (HTML tags).
    • Once you locate the node, you can access its attribute or text nodes.
  4. Working with Element Nodes:
    • Element nodes can be accessed using methods like getElementById() or querySelector().
    • getElementById() uses the value of an element's id attribute to locate the node.
    • querySelector() uses a CSS selector to find the first matching element.
  5. Modifying DOM Nodes:
    • You can modify the content of element nodes, including text and attributes, using JavaScript.
    • Attribute nodes can be modified using JavaScript to change class or id values.
  6. Traversal Methods:
    • Traversal methods involve moving through the DOM tree to locate specific nodes.
    • Examples of traversal methods include accessing child elements or navigating via the document node.
  7. Working with Text Nodes:
    • Once an element node is accessed, you can reach the text within and modify it.
    • Text nodes cannot have children.
  8. Steps for Working with DOM:
    • Locate the node that represents the element you want to work with.
    • Use its text content, child elements, and attributes.
  1. Events in Web Development:
    • Events refer to interactions or occurrences that happen in a web page, such as user actions or system events.
  2. Event Handling:
    • Event handling involves responding to events that occur on a webpage.
    • Two key concepts mentioned are Event Handlers and Event Listeners.
  3. Different Event Types:
    • Mouse Events: Events related to mouse interactions.
    • Keyboard Events: Events triggered by keyboard interactions.
    • Form Events: Events associated with form elements.
  4. Event Bubbling:
    • Event bubbling is a mechanism in which an event triggered at the deepest level of the DOM hierarchy is handled by ancestors of the target element.
    • This concept is crucial in understanding how events propagate through the DOM.
  5. Preventing Default Actions:
    • Some events have default actions associated with them (e.g., clicking a link navigates to a new page). You can prevent these default actions using techniques like preventDefault().

  1. Local Storage:
    • Local storage is a mechanism for storing data on the user's browser.
    • It provides a way to persistently store key-value pairs locally on the client side.
  2. Data Storage Formats:
    • The lesson mentions the use of JSON (JavaScript Object Notation) for working with local storage items.
    • JSON is commonly used for serializing and deserializing data in web development.
  3. localStorage API:
    • The localStorage API is introduced for interacting with local storage in JavaScript.
    • It provides methods for setting and getting key-value pairs.
  4. Cookies vs. Local Storage:
    • Cookies and local storage are mentioned in the context of data storage.
    • Local storage is a more modern and flexible alternative to cookies for storing larger amounts of data on the client side.
  5. Web Storage API:
    • The content refers to the Web Storage API, which encompasses both localStorage and sessionStorage.
    • This API provides a simple way to store and retrieve data on the client side.
